## 3. Voice & Tone

### Brand Personality

| Trait | We Are | We Are Not |
|-------|--------|------------|
| **Editorial** | We speak like a magazine caption. Precise, evocative, warm. | We don't sound like a SaaS dashboard or an AI chatbot. |
| **Understated** | We let the output speak. Minimal UI chrome. Quiet confidence. | We don't hype, shout, or use exclamation points. |
| **Attentive** | We remember what you like. We get it right the first time. | We don't make the user repeat themselves or babysit the tool. |
| **Premium** | Every interaction feels considered. No corners cut. | We don't feel cheap, rushed, or generic. |
| **Human** | We sound like a creative director, not software. | We never say "AI," "generate," "processing your request," or "optimizing." |

### Voice Testing

Before any user-facing copy ships, ask:
1. Does this sound like a creative director wrote it?
2. Would PhotoAI say this? (If yes, rewrite.)
3. Does it feel like a magazine caption or a software label?
4. Is there any trace of "AI product" language? (Must be zero.)

### Tone by Context

| Context | Tone | Example |
|---------|------|---------|
| **Onboarding** | Warm invitation | "Let's build your profile." |
| **The Brief** | Confident, precise | "You at a warm cafe table, morning light. 8 shots." |
| **Processing** | Quiet patience | "Processing your Drop..." |
| **The Edit ready** | Understated pride | "Your Edit is ready." |
| **Empty state** | Gentle nudge | "Your first Drop starts with a Scene." |
| **Error** | Calm, helpful | "Something didn't process. Try this Scene again." |
| **Payment** | Transactional, clean | "$10 · Single Drop. Processed via Stripe." |
| **Email (magic link)** | Personal, brief | "Here's your link to continue." |
| **Marketing landing** | Elevated, magazine | "Your photos, elevated." |

### Forbidden Words

Never use in user-facing surfaces:

- AI, artificial intelligence, machine learning, model
- Generate, generating, generation
- Prompt, prompting, prompt engineering
- Train, training, fine-tune, LoRA, embedding
- Render, rendering, compute, GPU, inference
- Algorithm, neural network, diffusion
- Optimize, optimized, enhancement
- Bot, assistant, agent, co-pilot
- Magic, wizard, miracle
- Processing your request, please wait, loading (use "Processing your Drop...")

## 6. Visual Identity — Provisional

> **Status:** The following visual decisions (colors, typography, imagery, spacing) are provisional recommendations based on the brand positioning. **Dani owns final design system.** These serve as a starting point for handoff.

### Design Direction

- **Editorial minimalism.** Think Kinfolk meets a boutique fashion journal. Generous whitespace. Single-column focus. No chrome.
- **Dark-first.** The product surfaces (Create, Brief, Edit) should default to a near-black ground with warm tints. This makes the generated photos feel like they're floating in a gallery, not trapped in a dashboard.
- **No AI visual cues.** No sparkle icons, no robot imagery, no gradient blobs, no "AI" badges. The interface should feel like a fashion editorial CMS, not a tech product.
- **Photo-forward.** Generated images should be the largest visual element on every screen they appear. UI chrome should recede.

### Color Palette (Provisional)

#### Primary Colors

| Name | Hex | Usage |
|------|-----|-------|
| **Near Black** | `#0a0a0a` | Primary text, primary surfaces (dark mode), buttons |
| **Warm White** | `#faf8f5` | Primary background (light mode), inverse text |

#### Accent Colors

| Name | Hex | Usage |
|------|-----|-------|
| **Camel** | `#c4946a` | Warm accent — CTAs, active states, hover highlights |
| **Sage** | `#8a9a8a` | Secondary accent — tags, secondary buttons, metadata |

#### Neutral Palette

| Name | Hex | Usage |
|------|-----|-------|
| **Background** | `#faf8f5` (light) / `#0a0a0a` (dark) | Page backgrounds |
| **Surface** | `#f0ede8` (light) / `#141414` (dark) | Cards, elevated surfaces |
| **Border** | `#d9d4cc` (light) / `#2a2a2a` (dark) | Dividers, borders |

| Error | `#c44d4d` | Errors, destructive actions |

### Typography (Provisional)

| Usage | Font | Weight | Size Desktop | Size Mobile | Line Height |
|-------|------|--------|--------------|-------------|-------------|
| Display / H1 | Cormorant Garamond | 400 | 64px | 40px | 1.1 |
| H2 | Cormorant Garamond | 400 | 40px | 28px | 1.2 |
| H3 | Inter | 500 | 24px | 20px | 1.3 |
| Body | Inter | 400 | 16px | 16px | 1.6 |
| Body Large | Inter | 400 | 18px | 18px | 1.6 |
| Caption / Meta | Inter | 400 | 12px | 12px | 1.4 |
| Label | Inter | 500 | 11px | 11px | 1.4 |

**Rationale:** Serif display (Cormorant) for magazine editorial energy. Clean sans (Inter) for interface pragmatism. The contrast between editorial serif and functional sans is the brand's core visual tension.

### Spacing & Radii (Provisional)

- **Density:** Generous. Editorial whitespace, not SaaS compactness.
- **Section spacing:** 80–120px between major sections.
- **Card padding:** 24px.
- **Radii:** Minimal. 2px for small elements, 4px for cards. Almost sharp — think print layout, not rounded SaaS.

### Imagery Style (Provisional)

- **In-product:** Generated photos dominate. No UI elements over them except save/discard actions on hover.
- **Marketing:** Real photography of creators in natural settings. Natural light. No studio flash. Skin texture visible. Slight film grain acceptable. The marketing photos should feel like the product's output.
- **No AI-generated imagery** in the product UI. Ever. If we need placeholder visuals, use real photography or solid color blocks.

## 7. Surface Pattern Guide

> **Status:** Interaction patterns are locked. Visual styling is provisional and owned by Dani.

### The Brief

- **Pattern:** Paragraph preview with inline clause editing.
- **Structure:** Centered, 640px max-width, single paragraph. No field labels. No form chrome.
- **Typography:** Body Large (18px), Inter 400. Each editable clause gets a subtle underline on hover. Click → inline input → Enter to confirm, Escape to cancel.
- **Behavior:** One Turn. After any edit, the paragraph updates. A gentle "Updated" fade appears. No save button — the text IS the confirmation.

### Scene Catalog (Browse)

- **Pattern:** Grid of editorial cards.
- **Structure:** Responsive grid, 2–4 columns. Each card is a tall vertical image with scene name and 2 tags below.
- **Typography:** Scene name in H3 (Inter 500, 24px). Tags in Label (Inter 500, 11px, uppercase, letter-spaced).
- **Behavior:** Click to select, card gets 1px camel border. Brief appears below without page transition.

### The Edit (Contact Sheet)

- **Pattern:** Dense grid of 8 variations.
- **Structure:** 4×2 grid on desktop, 2×4 on mobile. Uniform aspect ratio (4:5 for IG). Minimal metadata below each shot.
- **Typography:** None on the images. Caption below grid: "8 shots. Save your picks."
- **Behavior:** Hover → subtle overlay with save icon. Click → lightbox expand. Scroll → contact sheet scrolls as a unit.
